## Content Creation Squads

### Example 1: Instagram Carousel from Trending News

<Accordion title="Squad Overview" icon="instagram">
**Purpose:** Research trending tech news, generate multiple content angles, and produce Instagram carousel posts optimized for engagement.

**Performance Mode:** Alta Performance

**Formats:** Instagram Feed (Carousels)

**Agents:** 4
- 🔍 Pedro Pesquisa (Researcher)
- 💡 Ivan Ideia (Angle Generator)
- 📱 Iago Instagram (Carousel Creator)
- ✅ Renata Revisão (Quality Reviewer)

**Pipeline Steps:** 9
1. Research Focus (checkpoint)
2. Research (subagent)
3. Select Topic (checkpoint)
4. Generate Angles (inline)
5. Select Angle (checkpoint)
6. Create Carousel (subagent)
7. Approve Content (checkpoint)
8. Review (inline)
9. Final Approval (checkpoint)

**Key Features:**
- News-based content (user selects from trending stories)
- Angle generation creates 5 emotional perspectives per story
- Task decomposition: create → optimize → polish hooks (3 tasks)
- Veto conditions catch weak hooks and long slides before review
</Accordion>

<CodeGroup>
```yaml squad.yaml
squad:
  code: "tech-news-carousel"
  name: "Tech News Carousel Creator"
  description: "Researches trending tech topics and creates viral Instagram carousels"
  icon: "📱"
  version: "1.0.0"
  performance_mode: "alta"

skills:
  - web_search
  - web_fetch

data:
  - pipeline/data/research-brief.md
  - pipeline/data/domain-framework.md
  - pipeline/data/quality-criteria.md
  - pipeline/data/output-examples.md
  - pipeline/data/anti-patterns.md
  - pipeline/data/tone-of-voice.md

pipeline:
  entry: pipeline/pipeline.yaml
  steps:
    - step-01-research-focus.md
    - step-02-research.md
    - step-03-select-topic.md
    - step-04-generate-angles.md
    - step-05-select-angle.md
    - step-06-create-carousel.md
    - step-07-approve-content.md
    - step-08-review.md
    - step-09-final-approval.md
```

### Example 2: Multi-Platform Content Squad

<Accordion title="Squad Overview" icon="globe">
**Purpose:** Generate cohesive content across Instagram, LinkedIn, and Twitter from a single source (blog post, video, or news story).

**Performance Mode:** Alta Performance

**Formats:** Instagram Feed, LinkedIn Post, Twitter Thread

**Agents:** 6
- 🔍 Rita Research (Content Analyst)
- 💡 Adam Angle (Angle Generator)
- 📱 Ivy Instagram (Instagram Creator)
- 📝 Luna LinkedIn (LinkedIn Creator)
- 🐦 Tina Twitter (Twitter Creator)
- ✅ Victor Verdict (Multi-Platform Reviewer)

**Pipeline Steps:** 10
1. Source Input (checkpoint - user provides URL or uploads content)
2. Analyze Source (subagent - extract key insights)
3. Generate Angles (inline - 5 angles)
4. Select Angle (checkpoint)
5. Create Content (parallel subagents - Ivy + Luna + Tina)
6. Approve Content (checkpoint - review all 3 outputs)
7. Review (inline - Victor evaluates all platforms)
8. Final Approval (checkpoint)

**Key Features:**
- Parallel execution: 3 creators work simultaneously
- Cross-platform consistency (same angle, platform-specific execution)
- Single approval checkpoint for all outputs
- Reviewer applies both global and platform-specific criteria
</Accordion>

<CodeGroup>
```yaml squad.yaml
squad:
  code: "multi-platform-content"
  name: "Multi-Platform Content Creator"
  description: "Adapts single source into Instagram, LinkedIn, and Twitter content"
  icon: "🌐"
  version: "1.0.0"
  performance_mode: "alta"

skills:
  - web_search
  - web_fetch

data:
  - pipeline/data/research-brief.md
  - pipeline/data/platform-comparison.md
  - pipeline/data/cross-platform-quality-criteria.md
  - pipeline/data/output-examples.md
  - pipeline/data/tone-of-voice.md

pipeline:
  entry: pipeline/pipeline.yaml
  steps:
    - step-01-source-input.md
    - step-02-analyze-source.md
    - step-03-generate-angles.md
    - step-04-select-angle.md
    - step-05-create-content.md  # Parallel: Ivy + Luna + Tina
    - step-06-approve-content.md
    - step-07-review.md
    - step-08-final-approval.md
```


### Example 3: CSV Data Analysis and Insights

<Accordion title="Squad Overview" icon="chart-line">
**Purpose:** Analyze uploaded CSV data, identify patterns and anomalies, generate visualizations, and produce an executive report.

**Performance Mode:** Alta Performance

**Agents:** 4
- 📊 Dante Dados (Data Analyst)
- 📈 Ines Insight (Insight Generator)
- 📝 Rita Report (Report Writer)
- ✅ Vera Veredicto (Quality Reviewer)

**Pipeline Steps:** 8
1. Upload Data (checkpoint - user uploads CSV)
2. Analyze Data (subagent - statistical analysis, patterns, anomalies)
3. Generate Insights (subagent - interpret findings, identify trends)
4. Select Focus Areas (checkpoint - user chooses which insights to develop)
5. Create Report (subagent - write executive summary with visualizations)
6. Approve Draft (checkpoint)
7. Review (inline - validate accuracy, clarity, actionability)
8. Final Approval (checkpoint)

**Key Features:**
- Data-driven (no web research needed)
- Statistical analysis with anomaly detection
- Visualization generation (charts, graphs)
- Executive-friendly reporting
</Accordion>

## Automation Squads

### Example 4: Daily News Digest Automation

<Accordion title="Squad Overview" icon="newspaper">
**Purpose:** Monitor news sources daily, filter by relevance, summarize top stories, and send an email digest.

**Performance Mode:** Econômico (efficiency optimized for daily runs)

**Agents:** 4
- 🔍 Nancy News (News Monitor)
- ✂️ Sam Summarize (Summarizer)
- ✉️ Emily Email (Email Composer)
- 📬 Danny Deliver (Email Sender)

**Pipeline Steps:** 6
1. Configure Monitoring (checkpoint - first run only, sets topics and sources)
2. Fetch News (subagent - web search with time filter)
3. Summarize Stories (subagent - condense articles)
4. Compose Digest (subagent - format email with links)
5. Approve Draft (checkpoint - manual approval before sending)
6. Send Email (subagent - uses email skill)

**Key Features:**
- Scheduled execution (cron job or manual trigger)
- Econômico mode for efficiency (runs daily, lower token cost)
- Email skill integration
- Persistent configuration (topics stored in squad memory)
</Accordion>

<CodeGroup>
```yaml squad.yaml
squad:
  code: "daily-news-digest"
  name: "Daily News Digest"
  description: "Monitors news and sends daily email summaries"
  icon: "📰"
  version: "1.0.0"
  performance_mode: "economico"

skills:
  - web_search
  - web_fetch
  - sendgrid-email  # Custom MCP skill for sending emails

data:
  - pipeline/data/monitoring-config.yaml  # Topics, sources, recipients
  - pipeline/data/email-templates.md
  - pipeline/data/summarization-guidelines.md

pipeline:
  entry: pipeline/pipeline.yaml
  steps:
    - step-01-configure-monitoring.md  # First run only
    - step-02-fetch-news.md
    - step-03-summarize-stories.md
    - step-04-compose-digest.md
    - step-05-approve-draft.md
    - step-06-send-email.md
```

## Advanced Patterns

### Example 5: Multi-Squad Workflow (Chained Execution)

<Accordion title="Workflow Overview" icon="diagram-project">
**Purpose:** Chain multiple squads together for complex workflows.

**Use Case:** Content production pipeline
1. **Research Squad** - Gathers industry data and trends
2. **Content Squad** - Produces articles, posts, threads from research
3. **Distribution Squad** - Publishes content across platforms

Each squad's output becomes the next squad's input.
</Accordion>

<CodeGroup>
```bash Sequential Execution
#!/bin/bash
# run-content-pipeline.sh

# Step 1: Research
/opensquad run industry-research

if [ $? -ne 0 ]; then
  echo "Research squad failed. Aborting."
  exit 1
fi

# Step 2: Content Creation (reads research output)
/opensquad run content-production

if [ $? -ne 0 ]; then
  echo "Content squad failed. Aborting."
  exit 1
fi

# Step 3: Distribution (reads content output)
/opensquad run content-distribution

if [ $? -ne 0 ]; then
  echo "Distribution squad failed. Aborting."
  exit 1
fi

echo "✅ Full pipeline completed!"
```

```yaml content-production squad.yaml (reads research output)
squad:
  code: "content-production"
  name: "Content Producer"
  description: "Creates content from research squad outputs"
  icon: "✍️"
  version: "1.0.0"

pipeline:
  entry: pipeline/pipeline.yaml
  steps:
    - step-01-load-research.md  # Reads: squads/industry-research/output/latest/research-findings.md
    - step-02-generate-content-ideas.md
    - step-03-select-ideas.md
    - step-04-write-content.md
    - step-05-review.md
    - step-06-final-approval.md
```
</CodeGroup>

**Benefits:**
- Separation of concerns (each squad has one purpose)
- Reusable squads (run research squad independently)
- Easier debugging (isolate failures to specific squad)
- Parallel development (different teams work on different squads)
