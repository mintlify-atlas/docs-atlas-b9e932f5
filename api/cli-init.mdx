## Overview

The `init` command sets up OpenSquad in your project directory. It creates the necessary folder structure, installs core files, and walks you through an interactive setup wizard.

## Syntax

```bash
npx opensquad init
```

## What It Does

The init command performs the following steps:

1. **Checks for existing installation** - Detects if OpenSquad is already initialized and offers re-configuration
2. **Interactive setup wizard** - Prompts for user preferences:
   - Language selection (English, Português (Brasil), Español)
   - Your name for personalized outputs
   - IDE/tool selection (Claude Code, Antigravity, Codex, Cursor, VS Code + Copilot)
3. **Creates directory structure**:
   - `_opensquad/` - Core framework files
   - `_opensquad/_memory/` - Persistent memory and preferences
   - `_opensquad/_investigations/` - Content analysis data
   - `_opensquad/_browser_profile/` - Persistent browser sessions
   - `_opensquad/config/` - Configuration files
   - `_opensquad/core/` - Core system prompts and best practices
   - `skills/` - Installed skills directory
4. **Installs all available skills** - Automatically installs core skills from the registry
5. **Installs dependencies** - Runs `npm install` for project and dashboard
6. **Installs Playwright** - Sets up Chromium browser for web automation
7. **Writes preferences** - Saves your configuration to `_opensquad/_memory/preferences.md`

```

├── _opensquad/
```

## Interactive Prompts

### Language Selection

```
What language do you prefer for outputs?

  1. Português (Brasil)
  2. English
  3. Español

→
```

<Note>
The language selection affects all CLI output messages and is saved to your preferences.
</Note>

### Name Input

```
What's your name? →
```

Your name is stored and used for personalized content generation.

### IDE Selection

```
Which IDEs/tools do you use? (select all that apply)

  ◉ Antigravity
  ◯ Claude Code
  ◯ Codex (OpenAI)
  ◯ Cursor
  ◯ VS Code + Copilot
```

OpenSquad will install IDE-specific configuration files based on your selection:

- **Claude Code**: `.claude/system.md` and prompt files
- **Cursor**: `.cursorrules` and prompt files  
- **VS Code + Copilot**: `.github/prompts/` and `.vscode/settings.json`
- **Antigravity**: `.antigravity/rules.md`
- **Codex**: `AGENTS.md`

## Re-initialization

If OpenSquad is already initialized, running `init` again triggers re-configuration mode:

```bash
$ npx opensquad init

  🔄 Opensquad — Re-configure
```

This allows you to:
- Update your language preference
- Add/remove IDE configurations
- Reinstall core files without losing data

<Warning>
Re-initialization preserves your `_memory/`, `_investigations/`, and `squads/` directories.
</Warning>

## Preferences File

After initialization, your preferences are saved to `_opensquad/_memory/preferences.md`:

```markdown
# Opensquad Preferences

- **User Name:** John Doe
- **Output Language:** English
- **IDEs:** claude-code, cursor
- **Date Format:** YYYY-MM-DD
```

## Requirements

- **Node.js**: >= 20.0.0
- **npm**: Installed and accessible
- **Internet connection**: Required to download skills and dependencies
- **Write permissions**: In the target directory

## Common Issues

### Permission Denied

```bash
Error: EACCES: permission denied
```

Ensure you have write permissions in the target directory:

```bash
chmod u+w .
```

### Already Initialized

If you see the re-configuration prompt but want a fresh install, manually delete the `_opensquad/` directory first:

```bash
rm -rf _opensquad/
npx opensquad init
```

### Network Timeout

If skill downloads fail due to network issues, you can install them individually later:

```bash
npx opensquad install <skill-name>
```

## Next Steps

After initialization:

1. **Start using OpenSquad**:
   - In Claude Code: Type `/opensquad`
   - In Cursor: Type `/opensquad`
   - In VS Code Copilot: Type `/opensquad` in Chat (Ctrl+Alt+I)

2. **Create your first squad**:
   ```bash
   npx opensquad create
   ```

3. **Install additional skills**:
   ```bash
   npx opensquad skills
   ```
