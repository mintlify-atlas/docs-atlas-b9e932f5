This guide walks you through creating your first squad, approving its design, running it, and viewing the output.

## Prerequisites

Before starting, ensure you've completed [installation](/installation):

- ✅ Node.js 20+ installed
- ✅ OpenSquad initialized with `npx opensquad init`
- ✅ IDE configured (Claude Code, Cursor, etc.)

## Creating Your First Squad

### Step 1: Open the OpenSquad Menu

In your IDE, type:

```
/opensquad
```

You'll see an interactive menu:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  📘 OpenSquad Menu
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Create a new squad
2. Run an existing squad
3. My squads
4. More options
```

Select **"Create a new squad"** or directly use:

```
/opensquad create
```

### Step 2: Describe What You Need

The **Architect** agent will ask about your squad. Let's create a simple content squad:

```
Architect: What should this squad do? Describe the end result you want.

You: Create LinkedIn posts about AI trends
```

<Tip>
Be specific! Instead of "social media content", say "Instagram carousels about marketing tips" or "Twitter threads analyzing tech news".
</Tip>

### Step 3: Answer Discovery Questions

The Architect asks 3-4 questions to understand your needs:

<Steps>
  <Step title="Audience/Context">
    ```
    Architect: Who is this content for?
    1. Current customers
    2. Potential leads
    3. General audience
    4. Other
    
    You: 2
    ```
  </Step>
  
  <Step title="References (Optional)">
    ```
    Architect: Do you have any examples or reference profiles you'd like
    the squad to follow? (URLs to Instagram, LinkedIn, YouTube, Twitter/X)
    
    You: https://linkedin.com/in/example-profile
    ```
    
    If you provide URLs, the **Sherlock** investigator runs in the background to analyze real content patterns.
    
    <Note>
    You can skip references by typing "skip". The squad will use best practices instead of specific profile analysis.
    </Note>
  </Step>
  
  <Step title="Performance Mode">
    ```
    Architect: What quality level do you want for this squad?
    1. Alta Performance - Full pipeline with deep analysis, multiple formats,
       optimization tasks, and complete review (higher token cost)
    2. Econômico - Streamlined pipeline with basic analysis, main format only,
       and lightweight review (reduced token cost)
    
    You: 1
    ```
    
    **Alta Performance** creates 3-5 tasks per agent with dedicated optimization.
    
    **Econômico** creates 1-2 tasks per agent with embedded optimization.
  </Step>
  
  <Step title="Target Formats (Content Squads)">
    ```
    Architect: Which platforms/formats should this squad produce content for?
    1. LinkedIn Post - Text posts and document carousels
    2. LinkedIn Article - Long-form content
    3. Instagram Feed - Carousels and single image posts
    4. Twitter/X Thread - Multi-tweet threads
    5. Blog Post - Content marketing
    
    You: 1 2
    ```
    
    Select one or multiple numbers (space-separated).
  </Step>
</Steps>

### Step 4: Review the Squad Design

The Architect presents the proposed squad:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
I'll create a squad with 4 agents:

1. 🔍 Pedro Pesquisa — Researcher
   Discovers trending AI topics and analyzes engagement patterns
   Tasks: find-trends.md → rank-topics.md
   
2. ✍️ Luna LinkedIn — LinkedIn Content Creator
   Generates professional posts optimized for LinkedIn algorithm
   Tasks: create-linkedin-post.md → optimize-post.md
   Format: linkedin-post
   
3. 📝 Lia Longo — LinkedIn Article Writer
   Creates long-form thought leadership articles
   Tasks: create-article.md → optimize-article.md
   Format: linkedin-article
   
4. ✅ Renata Revisão — Quality Reviewer
   Ensures content meets brand standards and engagement criteria
   Tasks: score-content.md → generate-feedback.md

Pipeline:
[Research] → 🛑 Select Topic → [Creator: generate angles] → 
🛑 Select Angle → [Luna + Lia create content] → 🛑 Approve Content → 
[Review] → 🛑 Final Approval

Formats: linkedin-post, linkedin-article
Mode: Alta Performance

Reference materials:
- pipeline/data/research-brief.md
- pipeline/data/domain-framework.md
- pipeline/data/quality-criteria.md
- pipeline/data/output-examples.md
- pipeline/data/tone-of-voice.md

Does this look good?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

<Note>
Agent names use alliteration (same first letter) and are humorous puns in your chosen language. Each squad gets unique names.
</Note>

Type **"yes"** to approve, or request changes:

```
You: yes
```

### Step 5: Squad Generation

The Architect generates all squad files:

```bash

✓ Creating squad-party.csv

✓ Creating pipeline steps...

✅ Squad created: linkedin-ai-trends
📁 Location: squads/linkedin-ai-trends/

Ready to run? Type: /opensquad run linkedin-ai-trends
```

Your squad is now ready!

## Running Your Squad

### Execute the Pipeline

Run your newly created squad:

```
/opensquad run linkedin-ai-trends
```

The **Pipeline Runner** initializes:


## Viewing Output

### Output Directory Structure

Each squad run creates a timestamped folder:

```bash
squads/linkedin-ai-trends/
├── agents/
│   ├── pedro-pesquisa.agent.md
│   ├── luna-linkedin.agent.md
│   ├── lia-longo.agent.md
│   └── renata-revisao.agent.md
├── pipeline/
│   ├── pipeline.yaml
│   ├── data/
│   │   ├── research-brief.md
│   │   ├── quality-criteria.md
│   │   └── tone-of-voice.md
│   └── steps/
│       ├── step-01-research.md
│       ├── step-02-checkpoint-select-topic.md
│       └── ...
├── output/
│   └── 2026-03-13-143022/           # This run
│       ├── v1/
│       │   ├── research-findings.md
│       │   ├── linkedin-post.md
│       │   ├── linkedin-article.md
│       │   └── review-report.md
│       └── v2/                      # If you run revisions
├── _memory/
│   └── memories.md
├── squad.yaml
└── squad-party.csv
```

<Note>
**Version folders:** If you approve content for revision and the pipeline re-runs creation steps, outputs go to `v2/`, `v3/`, etc. This preserves all iterations.
</Note>

## Common Patterns

<AccordionGroup>
  <Accordion title="Creating a data analysis squad">
    ```
    /opensquad create "Squad that analyzes CSV data and generates insights"
    ```
    
    The Architect will create:
    - Data Analyst agent (reads CSV, performs analysis)
    - Insight Generator agent (identifies patterns)
    - Report Writer agent (produces executive summary)
    - Reviewer agent (validates accuracy)
    
    Pipeline: Upload Data → Analyze → Generate Insights → Write Report → Review
  </Accordion>
  
  <Accordion title="Creating a multi-platform content squad">
    ```
    /opensquad create "Instagram and Twitter content from YouTube videos"
    ```
    
    Provide YouTube channel URL as reference.
    
    The Architect creates platform-specific creators:
    - **Iago Instagram** - carousel creator
    - **Tiago Twitter** - thread creator
    
    Both work in parallel after angle selection.
  </Accordion>
  
  <Accordion title="Creating an automation squad">
    ```
    /opensquad create "Squad that monitors news and sends daily digests"
    ```
    
    The Architect designs:
    - News Monitor agent (web search + filtering)
    - Summarizer agent (condenses articles)
    - Email Composer agent (formats digest)
    - Distributor agent (sends via email skill)
    
    The squad can run daily via cron or manual trigger.
  </Accordion>
</AccordionGroup>

## Tips for Success

<Tip>
**Be specific in discovery questions.** Instead of "marketing content", say "LinkedIn thought leadership for B2B SaaS founders".
</Tip>

<Tip>
**Use reference profiles when possible.** Sherlock analysis extracts real patterns from successful content, making your squad much smarter.
</Tip>

<Tip>
**Start with Econômico mode** for your first squad to understand the flow. Upgrade to Alta Performance once you're comfortable.
</Tip>

<Tip>
**Provide feedback at checkpoints.** The squad learns from your approvals and rejections, storing preferences in `_memory/memories.md`.
</Tip>

<Warning>
**Don't skip the company profile.** If `_opensquad/_memory/company.md` is empty, OpenSquad will run onboarding first. This context personalizes all squad outputs.
</Warning>
